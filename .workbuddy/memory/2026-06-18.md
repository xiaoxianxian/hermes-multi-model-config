## 2026-06-18 16:47 - 聊天框切换模型报错修复

**新问题：** 在 Hermes 聊天框中通过 `/model` 切换模型后发送消息，又报 "No LLM provider configured"。

**根因 2：** 问题有两层：

1. **`config.yaml` 中 `model:` 字段是字符串格式**（`"agnes/agnes-2.0-flash"`），而 Hermes 的 provider 解析逻辑要求 `model:` 是**字典格式**才能检测到 custom provider。
2. Hermes 的 `select_provider_and_model()` 函数中，`config_provider = model_cfg.get("provider")` 只有在 `model_cfg` 是 dict 时才会生效。如果是字符串，`config_provider` 为 None，`effective_provider` 变成 `"auto"`，然后走到 `resolve_provider("auto")` 时找不到任何 provider，抛出 "No LLM provider configured"。

**修复 2：** 将 `config.yaml` 中的 `model:` 从字符串格式转换为字典格式：

```yaml
# 修改前
model: agnes/agnes-2.0-flash

# 修改后
model:
  default: agnes-2.0-flash
  provider: custom
  base_url: https://apihub.agnes-ai.com/v1
  api_key: sk-kG3MZLDkkDybT5ZOz02nAoXN6c2LviPzqcG8yirAPdcb9Va8
```

转换脚本：
```python
from hermes_cli.config import load_config, save_config

cfg = load_config()
model_str = cfg.get("model", "")
parts = model_str.split("/", 1)
provider_name = parts[0].strip()
model_name = parts[1].strip()

cfg["model"] = {
    "default": model_name,
    "provider": "custom",
    "base_url": f"https://{provider_name}/v1",
}

for cp in cfg.get("custom_providers", []):
    if cp.get("name") == provider_name:
        cfg["model"]["base_url"] = cp.get("base_url", "")
        if cp.get("api_key"):
            cfg["model"]["api_key"] = cp.get("api_key", "")
        if cp.get("api_mode"):
            cfg["model"]["api_mode"] = cp.get("api_mode", "")
        break

save_config(cfg)
```

**验证结果：**
- `config_provider` → `"custom"` (非 None)
- `effective_provider` → `"custom"` (非 "auto")
- `_active_custom_key_from_base_url()` 能匹配到 agnes provider
- `_has_any_provider_configured()` → `True`
