## 2026-06-18 16:47 - 聊天框切换模型报错修复

**新问题：** 在 Hermes 聊天框中通过 `/model` 切换模型后发送消息，又报 "No LLM provider configured"。

**修复 2：** 将 `config.yaml` 中的 `model:` 从字符串格式转换为字典格式。详见下方 `hermes-new-machine-guide.md` 配置步骤 3.2 中的转换脚本。

## 2026-06-19 11:00 - 优化配置脚本避免上下文丢失问题

**用户反馈：** Trae 和 WorkBuddy 切换模型后可以继续对话，不会丢上下文；但 Hermes 切换模型后需要重新输入才能继续。

**原因：** Trae 和 WorkBuddy 在切换模型时保持相同的 session ID 和对话历史，只是更换后端模型。而 Hermes 的 `_model_flow_named_custom()` 函数在切换模型时会保存 provider 和 base_url，但 `model:` 字段之前是字符串格式（`"agnes/agnes-2.0-flash"`），导致 Hermes 在 provider 解析链中无法识别 custom provider。

**根本原因：**
- Hermes 的 `select_provider_and_model()` 函数中，只有当 `model:` 是 **dict 格式**时，`config_provider = model_cfg.get("provider")` 才生效
- 如果 `model:` 是字符串，`config_provider` 为 None，`effective_provider` 变成 `"auto"`，然后 `resolve_provider("auto")` 找不到任何 provider
- 这就是切换模型后发送消息时报 "No LLM provider configured" 的原因

**修复方案：**
1. 修改 `setup-hermes-multi-model.sh`，不再需要先写入字符串再转换
2. 直接在步骤 4 创建 `model:` 为 dict 格式：
```yaml
model:
  default: agnes-2.0-flash
  provider: custom
  base_url: https://apihub.agnes-ai.com/v1
  api_key: YOUR_AGNES_KEY
```
3. 省去了原来的转换脚本（步骤 6），简化配置流程

**优势：**
- 配置后 `model:` 已经是 dict 格式，Hermes 能立即识别 provider
- 切换模型后发送消息不会报错
- 脚本更简洁，减少出错概率

2. 启用 fallback_model 配置（Agnes → DeepSeek 自动降级）
